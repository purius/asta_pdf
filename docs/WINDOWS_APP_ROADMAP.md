# PDF 뷰어/통합 도구 기능 확장 기획

## 목표

Windows에서 PDF를 빠르게 열고, 페이지를 정리하고, 여러 PDF를 합치고, A4 스캔/이미지 작업까지 처리하는 가벼운 로컬 PDF 도구로 발전시킨다. 인터넷 연결 없이 동작하고, 설치/제거/기본 앱 등록/오류 복구까지 Windows 데스크톱 프로그램 기준으로 완성도를 높인다.

## 원칙

- 기본 동작은 로컬 파일 기준으로 처리한다.
- PDF 원본은 사용자가 저장을 선택하기 전까지 덮어쓰지 않는다.
- 큰 PDF에서도 UI가 멈추지 않도록 렌더링과 파일 처리를 비동기로 분리한다.
- 기본 앱 등록, 우클릭 메뉴, 바로가기, 임시파일 정리는 설치/제거 흐름에서 일관되게 처리한다.
- 모든 위험 작업은 저장 경로를 명확히 보여주고 복구 가능한 방식으로 처리한다.

## 1단계: Windows 앱 기반 완성도

- 설정 창
  - PDF 우클릭 통합 메뉴 켜기/끄기
  - 현재 PDF 기본 앱 상태 표시
  - Windows 기본 앱 설정 열기
  - 최근 파일 개수 관리
  - 저장 접미사 기본값 관리
  - 임시파일 정리
  - 로그 폴더 열기
- 최근 파일
  - 파일 메뉴에 최근 PDF 표시
  - 존재하지 않는 파일 자동 제외
  - 최근 파일 전체 삭제
- 창 상태 저장
  - 마지막 창 크기와 위치 저장
  - 비정상적인 화면 밖 위치는 기본값으로 복구
- 로그
  - 앱 시작/종료, 주요 오류 기록
  - 문제 보고용 로그 폴더 접근
- 설치/제거 보강
  - 기본 앱 후보 등록
  - 파일 연결 명령 등록
  - 우클릭 메뉴 등록/제거
  - Shell association 갱신
  - 제거 시 등록 항목 정리

## 2단계: 안정성 및 복구

- 작업 상태 자동 백업
  - 열린 파일, 페이지 순서, 회전, 선택 상태 저장
  - 비정상 종료 후 복구 제안
- 오류 처리 개선
  - 암호 PDF, 손상 PDF, 접근 권한 오류 메시지 구분
  - 대용량 파일 처리 중 진행률 표시
- 임시파일 정책
  - 세션 임시파일 추적
  - 앱 종료/시작 시 오래된 임시파일 정리
  - 임시파일 정리 실패는 로그로만 기록

## 3단계: 대용량 PDF 성능

- 본문 렌더링 가상화
  - 현재 화면 주변 페이지만 캔버스 유지
  - 멀어진 페이지는 캔버스 해제 후 placeholder 유지
  - 빠른 스크롤 시 렌더 요청 취소
- 썸네일 지연 렌더링
  - 보이는 썸네일 우선 렌더
  - 백그라운드 큐로 나머지 렌더
  - 썸네일 캐시 상한 설정
- 렌더링 상태 표시
  - 페이지별 렌더 실패 표시
  - 전체 진행률 또는 상태바 표시

## 4단계: PDF 편집 기능 확장

- 선택 페이지 추출/저장
- 페이지 범위 저장
- 페이지 복제
- 빈 A4 여러 장 추가
- 이미지 파일을 PDF 페이지로 추가
- 클립보드 이미지 붙여넣기 위치/크기 조절
- 페이지 크기/회전 정보 표시
- PDF 정보 보기
  - 페이지 수
  - 파일 용량
  - 페이지 크기
  - 암호 여부
- 암호 PDF 열기

## 5단계: 사용성

- 다크모드
- 고DPI 배율 점검
- 메뉴 Alt 단축키 정리
- 키보드 포커스 표시
- 상태바 메시지 표준화
- 파일 드롭 성공/실패 안내 강화
- 시작 화면 개선
  - 최근 파일
  - 드롭 영역
  - 주요 작업 바로가기

## 6단계: 배포/업데이트

- 앱 버전 표시
- 릴리스 노트 표시
- 업데이트 확인
- 설치 파일 크기 최적화 검토
- WebView2 Runtime 안내

## 구현 순서

1. 설정/최근 파일/로그/기본 앱 상태
2. 자동 백업과 복구
3. 렌더링 가상화
4. PDF 편집 기능 확장
5. 다크모드/접근성/고DPI
6. 업데이트/배포 개선
